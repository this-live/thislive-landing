The problem it kills

Cross-agent memory loss + context rot

×

Context windows are fake memory

200K context is not 200K memory. Models degrade at the end of long contexts, every session resets, and you pay to re-process your own data.

×

Every agent re-discovers what the others already know

No shared context means duplicated work and one project leaking into another. Mnemos gives every agent the same scoped store — without sibling leakage.

The memory model

Scoped by hierarchy, curated by promotion

01

Container hierarchy

Memories live in paths like cortex/<pillar>. Recall on a container returns its memories plus ancestors' promoted memories — never siblings.

02

Scoped recall by default

Unscoped recall floors to a triage scope; it is never a global scan. Every result carries its source container tag.

03

Protected root + promotion

Direct writes to a shared root return 403. A memory reaches shared scope only via an explicit promotion ritual with an attestation.

04

Sessions + tombstones

Ephemeral session containers with TTL, atomic promote-on-close (exactly-once), and soft-delete tombstones that keep a version snapshot.
