Bryce Murad — Founder & AI Operator

Bryce Murad is the founder of This.Live, a local-first AI software publishing house. His current work centers on turning a fleet of machines, models, agent harnesses, and product teams into a practical operating system for building software with AI.

Current focus

Running and migrating active projects into MiniMax Code /team-backed workspaces, while keeping the public This.Live surface accurate: Cortex Suite, Beacon, Maestro, Signal & Noise, Digital Products Lab, Fieldhouse Games, FTAG Studio, DRAAN, SAQ/Storrs Aquatics support, Brokernomex/IQlume, Brainstorm Lab, and related fleet infrastructure.

Active project portfolio

  • Cortex Suite: local-first AI operating stack spanning Hermes runtime, Mnemos memory, Agent Fabric orchestration, Maestro routing, Forge adapter work, and personal/operator surfaces.
  • Beacon: public beta surface for local-service lead generation and demo-site opportunity workflows.
  • Maestro: model routing, consensus, quality labeling, and local/frontier model selection.
  • Signal & Noise: AI podcast pipeline with persistent memory, voice cloning, scripted research, and distribution automation.
  • Digital Products Lab: blog, short-video, Buffer/social, lead magnet, and operator-product engine for This.Live.
  • Fieldhouse Games: MiniMax team-backed sports-game studio for underserved mobile-first arcade sports titles.
  • FTAG Studio: Future Throwback Arcade Games, a non-sports game studio for original spiritual successors to Flash-era loops; first lane is Ages and Arrows.
  • Brokernomex / IQlume: staged revival of evidence-loop financial agents with paper, shadow, and bounded live-pilot gates.
  • SAQ / Storrs Aquatics: community/coaching support surface, website stability, swim-team tooling, and zero-disruption handoffs.

AI tools and coding harnesses

Daily operator across Cursor, Hermes, OpenClaw, Claude Code, Kimi Code, MiniMax Code, Codex, terminal-native agents, browser automation, direct provider APIs, delegated subagents, code review loops, and multi-agent workspace bootstrapping.

Model and AI systems knowledge

Hands-on evaluation and orchestration across OpenAI, Anthropic, Google/Gemini, Kimi, DeepSeek, Qwen, Gemma, Llama, MiniMax, Ollama, local models, frontier APIs, long-context systems, vision models, speech/voice systems, structured output, routing, consensus, quality labels, eval suites, and cost/performance tradeoffs.

Training, inference, and marketplaces

Works across Hugging Face, model marketplaces, local inference, vLLM, SGLang, Ollama, LoRA, adapter training, dataset curation, source-grounded evals, label harvesting, cloud training readiness, and local deployment paths to GPU-backed serving.

Infrastructure and deployment

Operates Railway, Cloudflare, Docker, nginx, GitHub, CI/CD, Tailscale, Terminus, SSH-first fleet operations, NAS-backed documentation, macOS/Linux administration, DNS/custom domains, webhook automation, cron jobs, browser tooling, vision/OCR fallback paths, incident response, and deploy verification.

Operating model

This.Live is built by pairing human product judgment with agent execution. Bryce defines the product direction, evidence gates, and quality bar; agents handle code, tests, docs, research, QA, migrations, and recurring operational loops. The goal is not generic automation. The goal is a working AI-native software company where projects accumulate memory, improve through receipts, and ship through real verification.
