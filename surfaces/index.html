The problem it kills

AI you rent, scattered across a dozen tenants

A chat app here, a voice assistant there, a terminal somewhere else — each a tenant on someone else's cloud, each with its own login, its own memory, its own bill. None of them know what the others did, and none of them are really yours.

A surface for every task, owned by someone else

Each app is a tenant on someone's cloud, your context held hostage in their silo. You're the integration layer, copy-pasting between tools that should share state — paying a separate rent to every one of them.

Centralized by default

The model, the memory, and the kill-switch all live on a server you don't control. When the provider changes the price, the policy, or the plug, your tools go with it — because you never owned them.

How it works

Four front doors onto your own fleet

Each surface is a different way into the same thing — a fleet of agents running on a backend you host. Not features in someone's app; doorways into a stack that's yours.

01

Pocket Agent

Speak, and your fleet acts. A live voice daemon turns spoken intent into dispatched work on the :19000 backend you host — pluggable voice backends, no cloud assistant in the loop.

02

Personal Life OS

A life-first front door onto the fleet, not a blank chatbox. PLO organizes the day around what you're doing and dispatches from it — and ships to Android via Capacitor, same backend.

03

Fleet Terminal

The raw door to the fleet: speech-to-text plus a local dispatch multiplexer, so you drive every node from one terminal — local, on hardware you own.

04

One desktop app

Every door in one window. An installable shell with a left activity rail, real files and terminal, and a live Voice console — the whole fleet, in one app you install and own.
