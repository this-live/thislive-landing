Niche Agents: Why the SAQ Bot Is One of My Best Builds

By Bryce Murad · this.live

Not every agent needs to be general purpose. Some of the most useful agents I have built are deeply niche, serving a specific community with specific knowledge. Let me tell you about the Storrs Aquatics agent, because it illustrates a pattern that I think applies to almost every organization on earth.

SAQ is my swim team's communication hub. Parents and coaches get Telegram access to an agent that knows everything about the team. Practice schedules. Meet results. Volunteer signups. Equipment orders. swimmer times. Warmup schedules. It answers questions at any hour. It never gets annoyed by the tenth parent asking the same thing. It never forgets a detail.

The agent is a Hermes instance running in a Docker container on Jarvis. It is isolated from the rest of my network. Its memory is scoped exclusively to swim team operations. It has skills for managing swim meet data, communicating with parents, and tracking team logistics. It knows Jaime is the head coach. It knows which swimmers are in which age groups. It knows the difference between short course and long course times.

This is not a chatbot that regurgitates a FAQ. It has access to documents, spreadsheets, and messages. It can look up specific swimmer times from last season. It can tell you what time warmups are for tomorrow's meet. It can help Jaime draft parent communications that sound human and informed. It can track volunteer commitments and remind people what they signed up for.

The point is that AI does not have to be a generic assistant. It can be a specialist. It can be embedded in a specific community, with specific knowledge, serving a specific purpose. The narrower the scope, the better the results. A generalist agent would struggle with swim team terminology. A niche agent thrives on it.

I am building SAQ because I care about that team. But the pattern applies everywhere. Every community, every business, every organization has repetitive communication and information retrieval needs. A niche agent solves those needs permanently. It becomes part of the organization's infrastructure.

A church could have an agent that knows the service schedule, the volunteer rotations, and the pastor's sermon themes. A gym could have an agent that knows class schedules, trainer availability, and membership details. A small law firm could have an agent that knows case statuses, client details, and filing deadlines.

Specificity is a feature, not a limitation. The more specific your agent's scope, the more value it delivers. Do not build one agent for everything. Build one agent for each thing that matters.

Your community will thank you.
