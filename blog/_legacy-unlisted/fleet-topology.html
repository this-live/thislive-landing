Fleet Topology: How My Agent Network Is Actually Laid Out

By Bryce Murad · this.live

People hear that I run multiple AI agents and they imagine a single server in a data center somewhere. That is not what I have built. My agent network spans six nodes across two states and multiple cloud providers. It is a distributed system by design, and understanding the topology matters because it explains why the system is resilient, fast, and secure.

Jarvis is my primary execution node. A headless Mac Mini running in my office. It handles everything from Docker containers to builds to deploys. It hosts Maestro, the Telegram bots, and most of the active agent workloads. If I need code written, tests run, or a service deployed, Jarvis does it.

Fred is a MacBook Pro. Product strategy, planning, and high-level thinking live here. It is the node I use when I need to reason through something complex without the noise of execution. Fred does not run crons that depend on heavy infrastructure. It thinks. It plans. It writes specs.

Atlas is another Mac Mini. Infrastructure management, monitoring, and network operations. It keeps the fleet healthy. It watches for issues, manages Tailscale, monitors services, and handles the operational side of running a distributed system.

Sparky is the DGX Spark. The GPU machine. It runs local inference with vLLM, hosts the larger models, and handles anything that needs serious compute. It also runs the Signal & Noise podcast pipeline. This is where the heavy lifting happens.

Bob is a Windows machine with a 5080. Creative work, gaming, and experimental tasks. It has its own WSL2 setup and runs Hermes independently. Bob is my creative lab. It is not production-critical, but it is part of the mesh.

NAS is a Linux storage node. Shared documents, memory archives, and the central repository for things that need to persist across the whole fleet. If every other node went down, the NAS would still have the data.

All of them are on Tailscale. They can all reach each other securely without exposing ports to the public internet. Maestro routes requests across the fleet based on which node has the right model, the right project scope, and the right availability. If Jarvis is overloaded, a request might route to Sparky. If Sparky's GPU is warm, a local inference job goes there.

This is not overkill. This is what a production agent network looks like when you are serious about it. Resilience through distribution. Speed through locality. Security through network isolation.

One machine is a point of failure. Six machines in a mesh is an organization.

That is the topology. That is the foundation.
