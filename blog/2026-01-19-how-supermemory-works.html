Memory is the first serious agent problem

By Bryce Murad · January 19, 2026

What I learned before Mnemos became the center

Agents without memory become strangers

The first thing that breaks when you work with agents every day is continuity. Not model quality. Not tool calling. Continuity. An agent can write a decent patch and still forget why the project exists by the next session. That gets old fast.

Early on I used Supermemory patterns to keep project context around. The useful idea was not magic recall. It was scoped recall. Beacon context should not bleed into podcast context. Personal context should not end up inside a public product. If memory is flat, the system gets sloppy.

Why I moved toward Mnemos

Mnemos became the direction because I wanted local control and cleaner project boundaries. I do not want memory to be a mystery service off to the side. I want it to behave like infrastructure I can inspect, route, back up, and constrain. Every project needs its own container. Every recall needs a reason.

That sounds boring until you have thirteen projects, multiple machines, and agents making decisions based on stale notes. Then boring becomes beautiful.

The rule now

The rule now is simple: memory should make the next action better without rewriting history. It should preserve durable facts, not every passing mood. It should know what project it is inside. It should help an agent ask fewer dumb questions while still leaving room for human judgment.

That is the memory layer I want under This.Live. Not a scrapbook. A nervous system.
