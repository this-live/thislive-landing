Cron and Automation: The Night Shift That Never Sleeps

By Bryce Murad · this.live

My agents do not wait for me to tell them what to do. A huge portion of their work happens while I am asleep. I wake up to a world that has already been updated, reviewed, and organized. It is like having a night shift that never gets tired, never calls in sick, and never forgets to do the thing you asked for.

The mechanism is cron jobs. Scheduled tasks that run at specific intervals. Some of my agents check for new emails every few hours. Others review my active git repositories overnight and leave code review notes for me in the morning. Some scrape news sources and summarize what happened in AI while I was sleeping. Some run health checks on my infrastructure and alert me if anything looks wrong.

These are not simple scripts. They are full agent sessions. The cron job spins up an agent with a specific prompt, gives it access to the relevant tools and memory scope, and lets it execute. The agent thinks. It makes decisions. It uses tools. It writes output. When it finishes, the results get delivered back to me via Telegram or written to a file I will see later.

The result is that I wake up to work that has already been done. My to-do list is updated. Stale tasks are cleared. News is summarized. Code is reviewed. Infrastructure is checked. It is a genuinely surreal experience the first time it happens. You roll over, check your phone, and your agent has already done three hours of work.

There is a heartbeat pattern too. Some agents poll periodically for things that need attention. If nothing needs attention, they stay quiet. If something does, they act or notify me. This prevents noise while ensuring nothing falls through the cracks. A heartbeat agent might check my calendar every hour and remind me fifteen minutes before a meeting. It might monitor a long-running process and tell me when it completes.

Automation is where agents stop being assistants and start being infrastructure. An assistant waits for you to ask. Infrastructure operates continuously, maintaining the system regardless of whether you are watching.

The magic is not in any individual cron job. It is in the composition. The agent that reviews code overnight feeds into the agent that updates my to-do list, which feeds into the agent that reminds me what to work on first. They are not isolated tasks. They are a pipeline of autonomous work that keeps the entire operation moving forward.

If you are running agents and you do not have automation, you are leaving the majority of their value on the table.

Hire a night shift. You will sleep better.
